## Overview

Agent Skills let coding agents like Claude Code and Cursor interact with Docent via the SDK. Instead of manually writing ingestion scripts or SDK queries, you describe what you want in natural language and the skill handles the implementation.

We currently offer two skills:

- [**Ingestion**](/agent-skills/ingestion): Upload existing agent logs to Docent without writing custom scripts. The skill detects your log format, validates the data, and handles the upload.
- [**Analysis**](/agent-skills/analysis): Run flexible analysis workflows by automatically writing DQL queries, comparing runs between models side-by-side at scale, taxonomizing failures into clusters, and more.

## Quick setup

<Note>
  We strongly recommend installing our Claude Code plugin so that you receive automatic updates.
</Note>

<Tabs>
  <Tab title="Claude Code (Recommended)">
    Run these commands inside Claude Code (not your terminal):

    Add the Transluce plugin marketplace:

    ```shell
        /plugin marketplace add transluce/claude-code-plugins
    ```

    Install the Docent you need:

    ```shell
        /plugin install docent@transluce-plugins
    ```

    Restart Claude Code after installation. Run `/help` to confirm the new commands are available.
  </Tab>
  <Tab title="Cursor / Other Harnesses">
    For Cursor, VS Code, and other agent harnesses, download the skill instructions and add them to your rules folder.

    1. Download [SKILL.md](https://github.com/transluce/claude-code-plugins/blob/main/plugins/docent/SKILL.md) from the plugin repository.
    2. Copy it to the `rules` folder:
       - **Cursor**: `.cursor/rules/docent.md`
       - **VS Code**: `.vscode/rules/docent.md`
       - If your harness does not support skills or rules, you can copy and paste SKILL.md as context.
    3. Follow the [Agentic Analysis](/agent-skills/analysis) setup instructions for MCP server configuration
  </Tab>
</Tabs>
